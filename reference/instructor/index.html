Instructor App APIs

This document provides a comprehensive overview of the APIs designed for the Instructor Application for CHITTI.

Content Planning Sheet (CPS)​

Get Content Planning Sheet​

Method: GET
Path: /course/:course_id/content-planning-sheet
Description: Retrieves the content planning sheet for a given course. Returns the sheet details along with associated remarks.
Path Params:
course_id: The ID of the course.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Response:

  • Success200
  • Not Found404
  • Unexpected Error500
{
  "message": "Content planning sheet fetched successfully.",
  "status": true,
  "data": {
    "course_id": "string",
    "content": "string",
    "status": "string",
    "deadline": "Date",
    "approved_by": ["string"],
    "remarks": [
      {
        "remark_id": "string",
        "remark": "string",
        "created_by": "string",
        "parent_id": "string",
        "created_at": "Date"
      }
    ]
  }
}
{
  "message": "Content planning sheet not found.",
  "status": false,
  "error_code": "cps/not-found"
}
{
  "message": "Couldn't fetch content planning sheet.",
  "status": false,
  "error_code": "cps/internal-server-error"
}

Update Content Planning Sheet​

Method: PATCH
Path: /course/:course_id/content-planning-sheet
Description: Updates the content planning sheet for a given course. If approve=true is passed in the query, it checks if all instructors have approved and updates the status to COMPLETED.
Path Params:
course_id: The ID of the course.
Query Params:
approve: true or false.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Request Body:

{
  "content": "string",
  "status": "string",
  "deadline": "Date",
  "approved_by": ["string"],
  "remark_ids": ["string"]
}

Response:

  • Success200
  • Unexpected Error500
{
  "message": "Content planning sheet updated successfully.",
  "status": true,
  "data": {
    "course_id": "string",
    "status": "COMPLETED"
  }
}
{
  "message": "Couldn't update content planning sheet.",
  "status": false,
  "error_code": "cps/internal-server-error"
}

Remarks (CPS & TCP)​

Get CPS Remarks​

Method: GET
Path: /course/:course_id/content-planning-sheet/remarks
Description: Retrieves the remarks associated with the content planning sheet of a course, including nested replies.
Path Params:
course_id: The ID of the course.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Response:

  • Success200
  • Not Found404
  • Unexpected Error500
{
  "message": "Remarks fetched successfully.",
  "status": true,
  "data": [
    {
      "remark_id": "string",
      "remark": "string",
      "created_by": "string",
      "parent_id": null,
      "replies": []
    }
  ]
}
{
  "message": "Course not found.",
  "status": false,
  "error_code": "course/not-found"
}
{
  "message": "Couldn't fetch remarks.",
  "status": false,
  "error_code": "remark/internal-server-error"
}

Create CPS Remark​

Method: POST
Path: /course/:course_id/content-planning-sheet/remark
Description: Creates a new remark on the content planning sheet and notifies instructors.
Path Params:
course_id: The ID of the course.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Request Body:

{
  "remark": "string",
  "parent_id": "string (optional)"
}

Response:

  • Success200
  • Invalid request400
  • Unexpected Error500
{
  "message": "Remark created successfully.",
  "status": true,
  "data": {
    "remark_id": "string",
    "remark": "string",
    "created_by": "string",
    "parent_id": "string"
  }
}
{
  "message": "Malformed payload. Required: remark",
  "status": false,
  "error_code": "payload/misformed-payload"
}
{
  "message": "Couldn't create remark.",
  "status": false,
  "error_code": "remark/internal-server-error"
}

Create TCP Remark​

Method: POST
Path: /course/:course_id/topic-creation-progress/:tcp_id/remark
Description: Creates a new remark on a TCP and updates resource statuses depending on the requesting user's role (INSTRUCTOR, STAFF, or EDITOR). Notifies relevant users.
Path Params:
course_id: The ID of the course.
tcp_id: The ID of the Topic Creation Progress.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Request Body:

{
  "remark": "string",
  "parent_id": "string (optional)",
  "role": "INSTRUCTOR | STAFF | EDITOR"
}

Response:

  • Success200
  • Invalid request400
  • Unexpected Error500
{
  "message": "Remark created successfully.",
  "status": true,
  "data": {
    "remark_id": "string",
    "remark": "string",
    "created_by": "string",
    "parent_id": "string"
  }
}
{
  "message": "Malformed payload. Required: remark",
  "status": false,
  "error_code": "payload/misformed-payload"
}
{
  "message": "Couldn't create remark.",
  "status": false,
  "error_code": "remark/internal-server-error"
}
