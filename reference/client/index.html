Student App APIs

This documentation provides an overview of the functionalities for APIs related to the Student Application.

Public APIs​

Health API​

Method: GET
Path: /health
Description: Verifies the health of the CHITTI. backend and acts as the primary source of verification if its an external failure of the backend as a monolothic layer is failing together.
Response Code: 200
Response Body: The backend is live 💓 - Chitti: Your last min prep buddy!!

Authentication​

Sign In​

Method: POST
Path: /auth/sign-in
Description: Authenticates a user and returns a Firebase custom token. Checks if the roll number exists, validates credentials, and handles device ID verification.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "X-FCM-Token": "string"
}

Request Body:

{
  "roll_number": "string",
  "password": "string"
}

Response:

  • Successful Sign in200
  • Invalid request400
  • Another Device Exists403
  • Unexpected Error500
    {
      "message": "Sign-in successful",
      "token": "string",
      "status": true,
      "data": {
        "name": "string"
      }
    }
    {
      "message": "Roll number is required",
      "status": false,
      "error_code": "payload/misformed-payload"
    }
    {
        "message": "Another device already exists, request for a revoke of your current device.",
        "status": false,
        "revokes_left": 3,
        "reset_token": "eyJhY2NvdW50X3Rva2VuIjo...",
        "error_code": "auth/device-exists"
    }

JWT for reset_token:

{
  "user_id": "221203XXXX", 
  "device_id_attempted": "new_device_uuid_9999",
  "exp": 1789945200 
}
    {
      "message": "Internal Server Error. Actual cause of the error",
      "status": false,
      "error_code": "auth/internal-error"
    }

Sign in With OAuth (Google/Apple)​

Method: POST
Path: /auth/oauth
Description: Authenticates a user with an external oauth provider like Google or Apple and returns a Firebase custom token. Checks if the user exists, validates credentials, and handles device ID verification. It also replaces the deprecated internal-login API as this API also works with Admins, Instructors and Staff.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "X-FCM-Token": "string"
}

Request Body:

{
  "provider": "string",
  "id_token": "string"
}

Response:

  • Successful Sign in200
  • Invalid request400
  • Another Device Exists403
  • User doesn't exist404
  • Unexpected Error500
    {
      "message": "Sign-in successful",
      "token": "string",
      "status": true,
      "data": {
        "name": "string"
      }
    }
    {
      "message": "ID Token is required",
      "status": false,
      "error_code": "payload/misformed-payload"
    }
    {
        "message": "Another device already exists, request for a revoke of your current device.",
        "status": false,
        "revokes_left": 3,
        "reset_token": "eyJhY2NvdW50X3Rva2VuIjo...",
        "error_code": "auth/device-exists"
    }

JWT for reset_token:

{
  "user_id": "221203XXXX", 
  "device_id_attempted": "new_device_uuid_9999",
  "exp": 1789945200 
}
    {
      "message": "User hasn't been signed-up on our platform.",
      "token":"string",
      "status": false,
      "error_code": "auth/user-not-found"
    }
    {
      "message": "Internal Server Error. Actual cause of the error",
      "status": false,
      "error_code": "auth/internal-error"
    }

Sign Up​

Method: POST
Path: /auth/sign-up
Description: Creates a new user account and returns student details.
Request Headers:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "X-FCM-Token": "string",
  "Authorization": "Bearer ey........"
}

Request Body:

{
  "name": "string",
  "semester": "string",
  "branch": "string",
  "organization": "string"
}

Response:

  • Successful Sign up200
  • Invalid request400
  • Unexpected Error500
    {
      "message": "Sign up successful",
      "status": true,
    }
    {
      "message": "Semester is required/Organization not supported yet.",
      "status": false,
      "error_code": "payload/misformed-payload"
    }
    {
      "message": "Internal Server Error. Actual cause of the error",
      "status": false,
      "error_code": "auth/internal-error"
    }

Revoke Device ID​

Method: PATCH
Path: /auth/device/reset
Description: Revokes a user's device ID, preventing them from logging in on that device.
Request Headers:

{
  "X-App-Version": "X.Y.Z"
}

Request Body:

{
  "reset_token": "eyJhY2NvdW50X3Rva2VuIjo..."
}

Response:

  • Successful Sign up200
  • Invalid request400
  • Revokes Exhausted403
  • User Not Found404
  • Unexpected Error500
    {
      "message": "XYZ's device ID has been revoked.",
      "status": true,
    }
    {
      "message": "Malformed payload",
      "status": false,
      "error_code": "payload/misformed-payload"
    }
    {
      "message": "No revokes left. Please request for a revoke.",
      "status": false,
      "error_code": "auth/revokes-exhausted"
    }
    {
      "message": "User not found.",
      "status": false,
      "error_code": "auth/user-not-found"
    }
    {
      "message": "Internal Server Error. Actual cause of the error",
      "status": false,
      "error_code": "auth/internal-error"
    }

[ADMIN] Fetch Token​

Method: POST
Path: /auth/token-exchange/:user_id
Description: Exchange a Firebase ID token for the given user ID. Should be only used by admins for token exchange.
Path Params:
user_id: The user id to generate the Firebase ID token

Request Headers:

{
  "X-App-Version": "X.Y.Z",
  "X-Internal-Secret": "ey...."
}

Response:

  • Successful Sign up200
  • Invalid request400
  • Unauthorized403
  • Unexpected Error500
    {
      "message": "Fetched Firebase ID token successfully",
      "status": true,
      "token": "ey..."
    }
    {
      "message": "Misformed payload",
      "status": false,
      "error_code": "payload/misformed-payload"
    }
    {
      "message": "Unauthorized",
      "status": false,
      "error_code": "auth/unauthorized"
    }
    {
      "message": "Internal Server Error. Actual cause of the error",
      "status": false,
      "error_code": "auth/internal-error"
    }
