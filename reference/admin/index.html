Admin Panel APIs

This document provides a comprehensive overview of the APIs designed for admin panel for CHITTI.

Revoke Requests​

Get Revoke Requests​

Method: GET
Path: /revoke-requests
Description: Fetch all pending revoke requests.
Request Header:

{
  "X-App-Version": "X.Y.Z",
  "X-Device-Id": "string",
  "Authorization": "Bearer ey..."
}

Response:

  • Success200
  • Unexpected Error500
{
  "message": "Revoke requests fetched successfully.",
  "status": true,
  "data": [
    {
      "request_id": 1,
      "roll_number": "string",
      "reason": "string",
      "status": "pending",
      "student": {
        "name": "string",
        "email": "string",
        "semester": 1,
        "roll_number": "string"
      }
    }
  ]
}
{
  "message": "Couldn't fetch revoke requests.",
  "status": false,
  "error_code": "revoke-requests/internal-server-error"
}

Accept Revoke Request​

Method: POST
Path: /revoke-requests/:request_id/accept
Description: Approves a pending device revoke request.
Path Params:
request_id: The ID of the request.
Response:

  • Success200
  • Invalid Request400
{
  "message": "Request accepted successfully.",
  "status": true
}
{
  "message": "Request already processed.",
  "status": false,
  "error_code": "request/already-processed"
}

Reject Revoke Request​

Method: POST
Path: /revoke-requests/:request_id/reject
Description: Rejects a pending device revoke request.
Path Params:
request_id: The ID of the request.
Request Body:

{
  "reason": "string"
}

Response:

  • Success200
  • Invalid Request400
{
  "message": "Request rejected successfully.",
  "status": true
}
{
  "message": "Reason is required.",
  "status": false,
  "error_code": "payload/missing-reason"
}

Storage Drive​

Get Signed URL​

Method: POST
Path: /drives/get-signed-url/:file_id
Description: Streams a private file directly from Google Drive.
Path Params:
file_id: The ID of the file in Google Drive.
Response: Returns the file stream directly with the appropriate Content-Type.

Initiate Course Migration​

Method: POST
Path: /drives/migration/course/:course_id
Description: Scans the database for topics ready to be migrated to R2 and generates migration tasks for them.
Path Params:
course_id: The ID of the course.
Response:

  • Success200
{
  "message": "All migration tasks have been generated.",
  "status": true,
  "data": [
    {
      "topic_id": "string",
      "topic_name": "string",
      "ready_resources": ["notes", "videos"],
      "migration_url": "string"
    }
  ]
}

Migrate Topic Resources​

Method: POST
Path: /drives/migration/course/:course_id/topic/:topic_id/migrate
Description: Initiates the migration of resources from Google Drive/Firebase to Cloudflare R2 via Server-Sent Events (SSE). Streams progress events.
Path Params:
course_id: The ID of the course.
topic_id: The ID of the topic.
Query Params:
resources: A comma-separated list of resources (e.g. notes,videos).
Response: Returns an SSE stream.
